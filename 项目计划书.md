# 寻遗记 · 项目计划书

> AI 非遗知识闯关游戏 · TRAE AI 创造力大赛 · 非遗创新赛道
> 版本：v1.0  ·  制定日期：2026-07-02  ·  初赛提交截止：2026-07-15

## 一、项目概述

### 1.1 一句话定位
「寻遗记」是一款**打开即玩、零安装、零注册**的 AI 非遗知识闯关 Web 游戏。用户在中国地图上选择省份、答题闯关、收集非遗图鉴，在 2 分钟一局的轻量游戏里，不知不觉记住家乡与全国的非遗知识。

### 1.2 核心价值主张
- **对用户**：把"需要专门去学的非遗知识"变成"随手可玩的趣味挑战"，2 分钟一局，无学习负担。
- **对社会**：用游戏化降低非遗传播门槛，AI 自动出题实现零运营成本覆盖全国 34 省份非遗知识。
- **对大赛**：纯前端 + 大模型 API 的轻量架构，TRAE 全栈开发，可复现、可传播、可扩展。

### 1.3 与原 demo 的关系
现有 HTML demo 已验证三件事：地图选省 → 答题闯关 → 图鉴收集的核心闭环成立；游戏化传播非遗的思路成立；纯前端 + 静态题库的技术路径成立。
本计划在此基础上做四项升级：
1. **内容扩展**：8 省份 → 34 省份（含港澳台）
2. **功能补全**：补齐原时间线已规划但未实现的道具、成就、分享、音效
3. **体验升级**：单文件 HTML → 组件化工程，支持动效、响应式、PWA
4. **设计刷新**：从金棕色调 → 墨黑/宣纸白 + 紫调光效的现代东方美学

## 二、目标用户与使用门槛

### 2.1 用户画像
| 画像 | 占比预估 | 核心诉求 |
|------|---------|---------|
| 通勤/午休消遣的年轻白领 | 40% | 2 分钟一局，有趣不费脑 |
| 中小学学生及家长 | 25% | 寓教于乐，可分享学习成果 |
| 文化旅游爱好者 | 20% | 系统了解各地非遗，做旅行功课 |
| 传统文化深度爱好者 | 10% | 题库深度 + 百科延伸阅读 |
| 大赛评委/媒体体验者 | 5% | 30 秒内看懂产品价值 |

### 2.2 使用门槛设计原则（面向大众的关键约束）
1. **零安装**：Web 链接即开即玩，支持 PWA 可"添加到桌面"但不强制
2. **零注册**：首次进入直接游玩，进度本地存储；仅在需要"云端存档/排行榜"时才可选昵称
3. **零学习成本**：3 步内上手（点省份 → 选答案 → 看结果），新手引导 < 10 秒
4. **单局轻量**：5 题/省，每题 15 秒思考空间，单局总时长 ≤ 2 分钟
5. **失败友好**：答错不惩罚，3/5 即过关；不过关可无限重试，题目随机不重复
6. **多端一致**：移动端优先（拇指可达），桌面端自适应，弱网可玩

## 四、技术架构

### 4.1 技术栈选型

| 层 | 技术 | 选型理由 |
|----|------|---------|
| 框架 | React 18 + TypeScript | 组件化便于功能扩展；类型安全降低 bug；TRAE 原生支持 |
| 构建 | Vite 5 | 启动快、HMR 流畅、产物小 |
| 样式 | Tailwind CSS + CSS Variables | 原子化提速；CSS 变量支撑主题切换 |
| 地图 | ECharts 中国地图 | 开箱即用、省界权威、交互成熟，省去自绘 SVG 成本 |
| 动效 | Framer Motion | 答题切换、点亮、图鉴翻牌等微动效 |
| 状态 | Zustand + localStorage 持久化中间件 | 轻量、无 boilerplate，天然适配本地存档 |
| 路由 | React Router | 地图 / 答题 / 图鉴 / 成就 / 百科多页切换 |
| AI 出题 | 大模型 API（经 TRAE 调用） | 离线批量生成题库 JSON；运行时仅用于 I1 老匠人对话 |
| PWA | vite-plugin-pwa | 离线缓存 + 可安装 |
| 部署 | Vercel / Netlify | 静态托管，自动 CI，全球 CDN |

### 4.3 数据模型（核心结构）

```ts
// 题目
interface Question {
  id: string;              // 题目唯一 id
  province: string;        // 所属省份
  category: HeritageCategory; // 非遗十大类别
  difficulty: 'easy' | 'normal' | 'hard';
  type: 'single' | 'multi' | 'judge';
  stem: string;            // 题干
  options: string[];       // 选项
  correct: number[];       // 正确答案索引
  explanation: string;     // 解析
  heritage_id: string;     // 关联非遗 id（用于跳转百科/图鉴）
  source?: string;         // 信息来源标注（质量控制）
}

// 非遗图鉴
interface Heritage {
  id: string;
  name: string;            // 非遗名称
  province: string;
  category: HeritageCategory;
  level: 'world' | 'national' | 'province'; // 保护级别
  icon: string;            // 镂空线性图标
  short_desc: string;      // 图鉴卡片简介
  long_desc: string;       // 百科详情
  rarity: 'common' | 'rare' | 'epic'; // 稀有度（图鉴收藏趣味）
}

// 用户存档
interface SaveState {
  lit_provinces: string[];          // 已点亮省份
  unlocked_heritages: string[];     // 已解锁图鉴
  props: { hint: number; fifty: number; skip: number; revive: number };
  achievements: string[];           // 已达成成就 id
  stats: { total_correct, total_answered, best_streak, ... };
  hometown?: string;                // 家乡省份
  nickname?: string;
  last_daily_date?: string;
  daily_streak: number;
}

// 非遗十大类别
type HeritageCategory =
  | '民间文学' | '传统音乐' | '传统舞蹈' | '传统戏剧' | '曲艺'
  | '传统体育、游艺与杂技' | '传统美术' | '传统技艺'
  | '传统医药' | '民俗';
```

### 4.4 AI 出题流程（离线批量生成，非运行时）

```
1. 采集：从官方非遗名录整理 34 省代表非遗清单（人工，1 天）
2. Prompt 模板：每省 10 题，覆盖十大类别，难度分级，附解析与来源
3. 批量调用：大模型生成 JSON 题库（经 TRAE 调用，1-2 天）
4. 人工校验：抽样 20% 校对事实准确性，错误项打回重生成
5. 入库：JSON 打包进前端静态资源
6. 运行时：前端随机抽题，不调用大模型，零延迟、零成本
7. 仅 I1 老匠人对话场景运行时调用大模型（低频、可降级为静态文案）
```

### 4.5 性能与离线策略
- 题库 JSON 按省份分片懒加载，首屏只加载用户当前选中省
- 图片资源 WebP + 懒加载，图标用 SVG sprite
- 动效遵守 `prefers-reduced-motion`
- PWA 缓存：app shell + 已访问省份题库，断网可继续游玩已加载内容

## 五、内容策略（题库）

### 5.1 题库规模
| 阶段 | 省份数 | 每省题数 | 总题数 | 备注 |
|------|--------|---------|--------|------|
| 初赛 MVP | 34 | 5 | 170 | 每省 5 题即可支撑闯关 |
| 决赛加强 | 34 | 10 | 340 | 每省 10 题，支持难度分级与重试不重复 |
| 长期 | 34 | 20+ | 680+ | 引入 UGC 持续生长 |

### 5.2 题目质量原则
1. **事实可核查**：每题标注信息来源（官方名录/联合国非遗名录）
2. **避免歧义**：4 个选项必须互斥，正确答案无争议
3. **解析有价值**：答错后解析要讲"为什么"，而非只标正确答案
4. **覆盖均衡**：每省题目覆盖该省 3-5 个不同非遗，避免只考最知名的 1 个
5. **难度合理**：简单题考查常识（如"京剧属于哪类"），困难题考查细节（如"某非遗列入联合国名录的年份"）

### 5.3 题库校验流水线
- AI 生成 → JSON Schema 校验 → 事实抽样人工校验 → 试玩测试 → 入库
- 错误题打回重生成，记录错误模式迭代 Prompt

## 六、UI/UX 设计方向

### 6.1 设计风格：现代东方美学
- **不要**：金棕色调、大红大绿、廉价国风贴图
- **要**：墨黑 + 宣纸白为底，紫调光效为魂，朱砂红仅用于关键节点（点亮/通过）的克制点缀
- 视觉灵感：水墨晕染、印章纹样、卷轴展开、镂空窗格

### 6.2 色彩规范
| 角色 | 色值 | 用途 |
|------|------|------|
| 主背景（深） | `#0E0E12` 墨黑 | 默认主题背景 |
| 主背景（浅） | `#F5F2EA` 宣纸白 | 浅色主题背景 |
| 主文字 | `#EDEAF3` / `#1C1917` | 深浅主题对应 |
| 强调主色 | `#7C5CFF` 紫调 | 与 TRAE 主页呼应，用于光效、点亮、CTA |
| 强调辅色 | `#A78BFA` 浅紫 | 悬停、渐变、辉光 |
| 朱砂点缀 | `#C73E3A` | 仅"已点亮/通过"等关键状态，克制使用 |
| 成功 | `#639922` 苔绿 | 答对反馈 |
| 错误 | `#E24B4A` 朱红 | 答错反馈 |

### 6.3 字体
- 标题：思源宋体 / 霞鹜文楷（文化质感）
- 正文：思源黑体 / PingFang SC（可读性）
- 数字：等宽字体（进度、计分对齐）

### 6.4 交互与组件原则
- **圆角非一致**：按组件形状调整圆角，地图区块用小圆角(8px)，卡片用中圆角(14px)，弹窗用大圆角(20px)
- **镂空图标**：全部使用线性/描边风格图标，禁用实心填充图标
- **非方形可点击**：省份用真实地图轮廓，答题选项用左圆右方异形卡片，道具用圆形按钮
- **动效有意义**：点亮用辉光扩散，答对用墨点晕开，图鉴解锁用卷轴展开——动效服务于"非遗"叙事，不为动而动
- **布局随形变化**：图鉴网格、地图、答题卡三类组件形状不同，各自布局逻辑独立，避免强行统一栅格

### 6.5 响应式与无障碍
- 断点：移动 < 640 < 平板 < 1024 < 桌面
- 移动端答题卡高度 ≥ 56px（拇指可达）
- 所有可点击元素 ≥ 44×44px
- 颜色对比度 ≥ WCAG AA
- 支持 `prefers-reduced-motion` 与 `prefers-color-scheme`

## 八、风险与对策

| 风险 | 影响 | 对策 |
|------|------|------|
| AI 出题事实错误 | 误导用户，损害公信力 | 每题标注来源；抽样 20% 人工校验；提供"报错"入口 |
| 34 省地图省界敏感 | 合规风险 | 使用 ECharts 官方地图数据，含港澳台，不自行绘制省界 |
| 移动端动效卡顿 | 体验下降 | 限制同屏动效数量；respect reduced-motion；低端机降级 |
| 运行时 AI 调用成本/延迟 | 仅 I1 受影响 | 题库全部离线；I1 老匠人对话可降级为预设文案 |
| 题库规模不足导致重复 | 重试体验差 | 每省 ≥ 5 题 + 随机抽题 + 难度分级；决赛扩到 10 题/省 |
| 大赛时间紧张 | 功能做不完 | 严格按 P0/P1 优先级砍功能；MVP 只做 P0，P1 时间允许才做 |
| 单人开发精力有限 | 全栈负担重 | TRAE AI 辅助编码；复用成熟库（ECharts/Framer/Tailwind）少造轮子 |
